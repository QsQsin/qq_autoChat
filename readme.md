# QQ 自动聊天机器人 (基于大模型 & NapCat)

本项目是一个功能强大的 QQ 个人消息自动回复辅助程序。它将传统的死板“自动回复器”升级为了接入大语言模型（如 DeepSeek、OpenAI 等）的智能体，不仅能模仿你本人的说话风格与朋友交流，还能根据聊天上下文氛围智能发送对应的表情包！

## ✨ 核心功能

1. **大语言模型驱动**：告别“人工智障”式的固定关键词回复。对接各大热门语言模型，实现理解上下文、通顺且流畅的自然语言自动回复。
2. **专属身份认知 (`persona.txt`)**：拥有一个可配置的“个人知识库”；模型会读取设定的身份、最近经历、口头禅和人设进行扮演，使其回复风格高度像你本人。
3. **表情包与颜文字系统**：
   - AI 回复时会自动加入 emoji 和颜文字，聊天更具感情。
   - **重磅功能**：支持根据“情绪”发送本地真实图片表情包（GIF、JPG）。AI 会判断聊天氛围并自主选择对应情绪分类（如：搞笑、无语、安慰）下的库中图片发送给对方，做到精准配图。
4. **图片消息感知**：自动解析好友发来的图片（将图片 `[CQ:image...]` CQ 码转化为 `[发来了一张图片]` 的文本事件），从而让纯文字模型做出像“我没空看图”或是自然过渡的反应，不会导致程序卡死或不知所云。

## 📌 环境要求与前置设定

- **Python**: 需要安装 Python 3.8 或以上版本。
- **环境依赖**:
  ```bash
  pip install fastapi uvicorn requests
  ```
- **NapCat (CQHTTP)**:
  本机需配置并运行 NapCat，提供基于 HTTP 协议的协议端通信。
  - **HTTP POST 上报地址 (回调地址)**: 设置向 `http://127.0.0.1:8080/` 投递消息。
  - **HTTP 发送 API**: 需开放端口并运行在 `http://127.0.0.1:3000` (配置于 `main.py` 的 `CQHTTP_API_URL`)。

## 📂 项目结构指南

- `main.py` - 服务核心运行代码。
- `persona.txt` - 个性化知识库配置文件，用自然语言写出你的个人信息和规矩。
- `memes/` - 本地表情包图库。其下必须“以情绪作为子文件夹名”分类：
  - `memes/开心/`
  - `memes/嘲讽/`
  - `memes/疑问/`
  (把平时常用的表情包对应放进去即可)

## 🚀 快速开始

### 1. 修改配置清单
打开 `main.py` , 根据你的实际情况进行信息修改：
- `AI_API_URL`：修改为你使用的大模型接口完整路径（例如 DeepSeek: `https://api.deepseek.com/chat/completions`）。
- `AI_API_KEY`：修改为你的 API 密钥。
- `AI_MODEL`：修改为调用的模型名字（例如 `deepseek-chat`）。
- `LISTEN_USER_QQ`：里面可以填入需要监听及回复的 QQ 号(如 `[123456, 987654]`)，若**留空 `[]` 则代表回复所有发来私聊消息的人**！
- *关于 Token:* 如 NapCat 启用了访问密钥，请确保替换 `send_reply` 函数头部（headers）的 Token 认证信息。

### 2. 完善性格设定
打开或新建 `persona.txt`，把你的个人属性加进去。大模型将会读取这个文件扮演你：
```txt
姓名：张三
职业：后端程序员
口头禅：啊这，确实，太草了
要求：说话简短，多用语气词。如果被问到没了解过的事情就直接说不知道。最近我在忙着写论文。
```

### 3. 配置智能表情包 （可选但强烈推荐！）
在项目根目录新建 `memes` 文件夹，接着在内部新建不同“分类主题”的子文件夹，将对应的图片扔进去。
例如：
```
memes/
├── 震惊/
│   ├── what.jpg
│   └── omg.gif
├── 搞笑/
│   └── laugh.png
└── 骂人/
    └── getout.jpg
```
重启程序后，大模型即可感知到这些文件夹，并在合适的氛围下调用 `[表情包:震惊]` 将真实图片发给好友。

### 4. 运行服务
进入对应目录执行：
```bash
python main.py
```
出现 `🚀 强大版的 QQ 机器人监听服务已经连线启动 (端口 8080)！` 字样即可。接着你大可让朋友发消息测试你的 AI 替身了！
